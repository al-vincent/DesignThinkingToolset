Limitations on images

There are two main limitations on the images that SNIP can handle:
  • The image format must be JPEG, PNG or Bitmap.
  • The image cannot be more than 4MB in size.

This is because SNIP uses Machine Learning services provided by Microsoft Azure. The Azure services offer very good performance, but unfortunately they also have constraints on the images they can handle.

These limitations may also change over time; if so, this will be reflected in SNIP.

If you're using an image that's larger than 4MB, there are a few options:
  • If your image is a PNG or Bitmap, save a new version as a JPEG and try that. This will compress the image, often by quite a lot (especially for Bitmaps). You'll lose a little bit of fidelity, but it will make very little difference to SNIP.
    Note that this won't work if your image is already a JPEG.
  • Crop the image to remove anything that doesn't need to be analysed before you upload it. This can remove quite a lot of unnecessary space from the image, reducing the size by quite a lot.
  • Reduce the fidelity of the image, using image editing software. This doesn't have to be complex; simple tools like Windows Paint will resize an image very easily, and making it smaller will reduce the fidelity (and file-size).
  • If none of the methods above work, you can break the image into multiple smaller images, and run each of these through SNIP. Again, breaking the image up is straightforward using a simple image editor like Paint.
